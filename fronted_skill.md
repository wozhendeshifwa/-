## 前端美学设计技能

本技能指导创建独特的、生产级前端界面，避免通用的"AI 同质化"美学。实现真正可运行的代码，并对美学细节和创意选择给予极高的关注。

### 输入

用户提供前端需求：要构建的组件、页面、应用或界面。可能包含关于用途、受众或技术约束的背景信息。

### 设计思维

在编写代码之前，理解上下文并致力于一个**大胆的美学方向**：

- **目的**：这个界面解决什么问题？谁在使用它？
- **基调**：选择一个极端方向——极简主义、极繁主义混沌、复古未来主义、有机/自然、奢华/精致、 playful/玩具感、编辑/杂志风格、粗野主义/原始、装饰艺术/几何感、柔和/粉彩、工业/实用主义等等。有很多风格可供选择。可以将这些作为灵感，但设计要忠于所选的美学方向。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异化**：什么让这个界面**令人难忘**？人们会记住哪一件事？

**关键**：选择一个清晰的概念方向，并精确地执行它。大胆的极繁主义和精致的极简主义都有效——关键在于**有意为之**，而非强度。

### 实现

然后实现可运行的代码（HTML/CSS/JS、React、Vue 等），要求：

- 生产级且功能完备
- 视觉上引人注目且令人难忘
- 具有清晰美学观点的连贯性
- 每一个细节都经过精心打磨

## 前端美学指南

关注以下几点：

### 字体排印

选择**美观、独特且有趣**的字体。避免使用 Arial 和 Inter 等通用字体；转而选择能提升前端美学的独特选择——出人意料、富有特色的字体选择。将独特的展示字体与精致的正文字体搭配使用。

### 色彩与主题

致力于一个**连贯的美学**。使用 CSS 变量保持一致性。主导色彩搭配锐利的强调色，效果优于平淡、均匀分布的调色板。

### 动效

使用动画来实现效果和微交互。对于 HTML 优先选择纯 CSS 解决方案。对于 React 使用 Motion 库（如果可用）。专注于高影响力的时刻：**一次精心编排的页面加载，配合错开的元素呈现（使用 `animation-delay`）**，比分散的微交互能带来更多愉悦感。使用滚动触发的效果和令人惊喜的悬停状态。

### 空间构图

出人意料的布局。不对称。重叠。对角线流动。打破网格的元素。慷慨的留白或可控的密度。

### 背景与视觉细节

创造氛围和深度，而不是默认为纯色。添加与整体美学相匹配的情境效果和质感。运用富有创意的形式，如渐变网格、噪点纹理、几何图案、分层透明度、戏剧性的阴影、装饰性边框、自定义光标和颗粒叠加层。

### 禁止使用

**绝不**使用通用的 AI 生成美学，例如：

- 过度使用的字体系列（Inter、Roboto、Arial、系统字体）
- 陈词滥调的配色方案（特别是白色背景上的紫色渐变）
- 可预测的布局和组件模式
- 缺乏上下文特定特征的千篇一律的设计

## 核心理念

发挥创造力，做出**出人意料**的选择，让人感觉真正是为该上下文而设计。没有两个设计应该是相同的。在深色和浅色主题、不同字体、不同美学风格之间变化。**不要**在不同生成中趋同于常见的选择（例如 Space Grotesk）。

**重要提示**：将实现复杂度与美学愿景相匹配。极繁主义设计需要复杂的代码，包含大量的动画和效果。极简主义或精致的设计则需要克制、精确，以及对间距、排版和微妙细节的仔细关注。优雅来自于很好地执行愿景。

**记住：Cursor 能够完成非凡的创意工作。不要有所保留，展示当跳出框框思考、全身心投入到一个独特愿景中时，真正能够创造出什么样的作品。**
